    2.3 框架要做到良好的Tree-Shaking

    AzureTimez 2023/6/9 Vue3

      上一章聊到设置常量，就不会去执行调试代码，但是这部分代码不会被执行，那么就需要把他们在编译时，将他们删除。在Vue.js来举例子，例如<Transition>组件，如果我们的项目中根本没有用到该组件，那么它就 不会存在于最终构建资源中。Vue中主要使用的就是Tree-Shaking。

      Tree-Shaking，在前端领域，这个概念因rollup.js而普及。简单地说，Tree-Shaking指的就是消除那些永远不会被执行的代码，也就是排除dead code，现在无论是rollup.js还是webpack，都支持Tree-Shaking。

      想要实现Tree-Shaking，必须满足一个条件，即模块必须是ESM（ES Module），因为Tree-Shaking依赖ESM的静态结构。以下是以rollup.js为例，看看Tree-Shaking如何工作，目录结构如下：

      |-- demo
|   |- package.json
|   |- input.js
|   |- utils.js

      3
      4

      yarn add rollup -D
# 或者 npm install rollup -D

      // input.js
import { foo } from './utils.js'
foo()
// utils.js
export function foo(obj) {
  obj && obj.foo
}
export function bar(obj) {
  obj && obj.bar
}

      这句命令的意思是：以input.js文件为入口，输出ESM，输出的文件叫做bundle.js。

      // bundle.js
function foo(obj) {
  obj && obj.foo
}
foo();

      可以看到已经不包含bar函数，这说明Tree-Shaking起作用了，将bar当做dead code删除了。同时，我们发现foo其实也没有任何作用，但是为什么没有作为dead code删除呢。
      这就涉及到Tree-Shaking中的第二个关键点——副作用。副作用：当调用函数的时候会对外部产生影响，例如修改了全局变量。
      想要静态地分析哪些代码是dead code很有难度。

      因为静态分析JavaScript代码很困难，所以像rollup.js这类工具都会提供一个机制，让我们能明确地告诉rollup.js，“放心吧，这段代码不会产生副作用，你可以移除它”。

      import { foo } from './utils'
/*#__PURE__*/ foo()
      1
      2

      加过以后，再进行编译，我们就会发现foo消失了。Vue.js 3使用了大量/*#__PURE__*/注释，这个注释不仅可以作用于函数，可以用到任何语句上。webpack以及压缩工具（如terser）都能识别到它。
