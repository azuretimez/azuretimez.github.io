    • 性能优化进阶

    # 性能优化进阶

    # Navigation Time API (**)

    1. navigationStart 表示的是上一个文档卸载结束的时间戳

    2. unloadEventStart / unloadEventEnd 完成回调 / 第一个重定向开始前

    3. redirectStart / redirectEnd 第一个重定向发生时间 / 最后一个重定向完成时间 => 衡量重定向发生的时间

    4. fetchStart 准备开始下载文档（检查缓存之前）

    5. domainLookupStart / End <=> connectStart / End HTTP开始或者重新建立的时间 vs TCP连接所需要的时间

    6. requestStart / End < = > reponseStart / End

    7. dom - Loading / Interactive / ContentLoadedEventStart / ContentLoadedEventEnd / Complete 开始渲染dom时间 / 完成解析DOM树、并没有开始加载依赖 / 页面内资源加载时间 / dom树解析完成

    8. loadEventStart / End 文档加载load事件回调 开始 到 结束

        // index.html
    <script>
        javascript: (() => {
            const perfData = window.performance.timing;
            const pageLoadTime = perfData.loadEventEnd - perfData.navigationStart;
            console.log('页面跳转加载耗时：', pageLoadTime, "ms");
        })();
    </script>

    # Core Web Vitals - 网页核心性能指标

    • 每个方面代表用户体验的不同方向 => 可衡量、反应真实感受：加载、交互性、视觉稳定性
    # LCP => 页面的装载性能 => 前2.5s内进行最大内容渲染

    a. 最大内容有哪些？

    • 通过url加载的背景图片元素
    • 主要文本节点及其子集的块级元素

    b. 影响LCP的原因？

    • 服务器响应
    • 阻断渲染的js和css
    • 资源过大
    • 客户端机器

    c. 针对性改造

    • 服务器优化 减少对于资源的依赖 => 强缓存、协商缓存

    • 减少资源阻断渲染 后置逻辑

    • 资源问题 => 格式、压缩、云管理、CDN

    • 客户端机器 => 优雅降级

    # FID:

    衡量交互性

    • 页面首次输入延迟应当小于100ms

    a. 减少JS执行时间

    • 缩小压缩JS文件
    • 延迟加载不需要的JS => SPA
    • 减少未使用的polyfill

    b. 分解耗时任务

    • 定义长任务 => 异步拆解功能拆分

    c. worker

    • JS Worker: Web Worker, service Worker, Worklet
        // 1. web worker
    // main.js
    // 新增worker
    const myWorker = new Worker('worker.js');

    // 与main thread之间通信

        console.log(e.data);

    // worker.js
    self.onmessage = function(e) {
        console.log(e.data);
        self.postMessage(result);

    // 2. service worker
    // main.js
    navigator.serviceWorker.register('/service-worker.js');

    // service-worker.js
    self.addEventListener('install', function() {});
    self.addEventListener('activate', function() {});
    self.addEventListener('fetch', function() {
        event.respondWith(
            Caches.match(event.request);
        )
    });

    // 3. worklet
    // main.js
    CSS.paintWorklet.addModule('myWorklet.js');

    // myWorklet.js

    // app
